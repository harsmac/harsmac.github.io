Diffusion Models: The Complete Story

From Noise to Images — Sharp Like GANs, Stable Like VAEs

Harshitha Machiraju · 18 min read
Part I

Why Diffusion?

1 VAE vs GAN: The Tradeoff We Want to Break

ModelTrainingOutput QualityThe Problem
VAEStable ✓Blurry ✗MSE loss encourages hedging
GANUnstable ✗Sharp ✓Adversarial game is fragile

Can we get both? Sharp outputs AND stable training?

2 The Diffusion Insight: Gradual Refinement

GAN: random noise z ──→ [Generator] ──→ full image One network must learn the ENTIRE mapping from noise to image. Huge leap. Easy to mess up. Unstable. Diffusion: pure noise ──→ slightly ──→ less ──→ ... ──→ almost ──→ clean cleaner noisy clean image Many small steps. Each step is a tiny correction. Much easier to learn. Stable.
Key Insight

Instead of generating an image in one shot, gradually refine from noise. Each step only needs to make a small correction — much easier than generating everything at once.

Part II

Setup & Nomenclature

3 The Full Notation

SymbolNameWhat It Is
$x_0$Clean imageOriginal, noise-free image
$x_t$Noisy image at step tImage after adding noise for t steps
$x_T$Pure noiseAfter T steps (typically 1000), complete Gaussian noise
$\epsilon$The noiseGaussian noise $\sim \mathcal{N}(0, I)$
$\bar{\alpha}_t$Cumulative signal retention$\prod_{s=1}^{t} (1-\beta_s)$ — total signal remaining
$\epsilon_\theta(x_t, t)$Noise predictorNeural network that predicts the noise in $x_t$

4 The Forward Process: Adding Noise

The forward process is fixed (not learned). We can jump directly to any timestep:

$$x_t = \sqrt{\bar{\alpha}_t} \, x_0 + \sqrt{1-\bar{\alpha}_t} \, \epsilon$$

This says: $x_t$ is a weighted combination of the original image and noise.

  • $\sqrt{\bar{\alpha}_t}$ = how much original image remains
  • $\sqrt{1-\bar{\alpha}_t}$ = how much noise is added
t=0: 🐱 (clean cat) t=100: 🐱 + slight grain t=500: vague blob + lots of noise t=1000: pure static (can't tell it was a cat)

5 The Reverse Process: What We Learn

The reverse process goes from noisy to clean. This is what we train.

🤔 Question

"So the network directly predicts the less-noisy image?"

✓ Answer

It could! But there's a better way. Instead of predicting the image directly, we predict the noise $\epsilon$. This works much better.

Part III

The Training Objective

6 Why Predict Noise?

Recall how we made $x_t$:

$$x_t = \sqrt{\bar{\alpha}_t} \, x_0 + \sqrt{1-\bar{\alpha}_t} \, \epsilon$$

If we can predict $\epsilon$, we can recover $x_0$:

$$x_0 = \frac{x_t - \sqrt{1-\bar{\alpha}_t} \, \epsilon}{\sqrt{\bar{\alpha}_t}}$$

Predicting $x_0$

"Look at this noisy mess. What's the original cat?"

Hard when $t$ is large (pure noise).

Predicting $\epsilon$

"What noise was added?"

Same difficulty at all timesteps.

7 The Loss Function

$$\mathcal{L} = \mathbb{E}_{t, x_0, \epsilon}\left[ \| \epsilon - \epsilon_\theta(x_t, t) \|^2 \right]$$

In words: "Predict the noise that was added. Minimize prediction error."

def train_step(model, x_0): # 1. Random timestep t = randint(1, T) # 2. Sample noise epsilon = torch.randn_like(x_0) # 3. Create noisy image (one-step formula) alpha_bar_t = alpha_bar[t] x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * epsilon # 4. Predict the noise epsilon_pred = model(x_t, t) # 5. Simple MSE loss loss = F.mse_loss(epsilon, epsilon_pred) return loss

8 Why Doesn't MSE Cause Blur Here?

🤔 Question

"Wait — MSE caused blur in VAE. Why doesn't it cause blur here?"

✓ Key Insight

VAE predicts images. Diffusion predicts noise.

VAE predicting images: "Should this pixel be cat-whisker or background?" MSE: "Output 0.5 gray to hedge" Result: BLUR Diffusion predicting noise: "What Gaussian noise was added to this pixel?" Answer: UNIQUE. There's no averaging across outcomes. The noise ε is fixed — we just need to identify it. Result: NO BLUR
Key Insight

Noise is not ambiguous like semantic content. Given $x_t$ and $t$, there's ONE correct $\epsilon$ that satisfies the forward equation. No hedging needed.

Part IV

Architecture & Extensions

9 The U-Net Architecture

The noise predictor $\epsilon_\theta(x_t, t)$ is typically a U-Net:

Input: x_t (noisy image) ↓ ┌─────────────────────────────┐ │ Downsample blocks │ (encode) │ 64 → 128 → 256 → 512 │ └─────────────────────────────┘ ↓ ┌─────────────────────────────┐ │ Middle block │ (self-attention) └─────────────────────────────┘ ↓ ┌─────────────────────────────┐ │ Upsample blocks │ (decode + skip connections) │ 512 → 256 → 128 → 64 │ └─────────────────────────────┘ ↓ Output: ε̂ (predicted noise)

Key addition: time embedding. The network needs to know which timestep $t$ we're at (like positional encoding in transformers).

10 Classifier-Free Guidance

How do we control what gets generated? Add a condition $c$ (text, class label, etc.):

$$\hat{\epsilon} = \epsilon_\theta(x_t, t, \emptyset) + s \cdot (\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, \emptyset))$$

where $s > 1$ is the guidance scale. "Push harder in the direction that matches the condition."

11 Latent Diffusion (Stable Diffusion)

A 512×512 image has 786,432 pixels. Running diffusion in pixel space is expensive.

Standard Diffusion: noise ←→ diffusion ←→ image (512×512×3) (expensive: huge tensors) Latent Diffusion: image → VAE encoder → latent (64×64×4) ↓ diffusion here! (cheap) ↓ VAE decoder → image
Key Insight

Latent diffusion gets 64× speedup (going from 512×512 to 64×64). The VAE handles low-level details; diffusion handles high-level structure. This is what Stable Diffusion does.

12 Diffusion Transformers (DiT)

U-Net was the original architecture. But transformers scale better.

DiT Architecture: 1. Patchify: split latent into patches (like ViT) 2. Linear embed each patch → tokens 3. Add positional + time embedding 4. Process with transformer blocks 5. Unpatchify → predicted noise

DiT powers Sora, Stable Diffusion 3, and other cutting-edge models.

13 Final Comparison: VAE vs GAN vs Diffusion

The Tradeoff: Diffusion is slow. Generating one image needs ~50-1000 forward passes.

Fixes:

  • DDIM: Deterministic sampling, fewer steps (~50)
  • Latent diffusion: Work in smaller latent space
  • Consistency models: Distill to single-step generation
🎯 Interview Narrative

"VAE has blurry outputs because MSE on images encourages hedging. GAN fixes blur with adversarial training but is unstable and mode-collapses.

Diffusion takes a different approach: gradually add noise to images (forward process), then train a network to predict and remove that noise (reverse process). Each step is a small correction, much easier than generating an image in one shot.

The key insight: predict the noise ε, not the image directly. Noise is simple (Gaussian), so MSE works fine — no hedging about semantic content.

Training is simple: sample image, add noise at random timestep, predict the noise, MSE loss. No adversarial game.

Sampling is slow (many steps), but latent diffusion fixes this by working in compressed VAE latent space. Modern models use transformer backbones (DiT) instead of U-Net for better scaling."
